# Brand Voice Rubric

Score each criterion 1 to 5. Copy requires a score of 4 or higher on each criterion to pass.

## 1. Technical Grounding
Technical details support verifiable claims.

| Score | Description |
| :--- | :--- |
| 5 | Includes specs, architecture, and measurable outcomes. |
| 4 | Features concrete claims with clear meaning. |
| 3 | Combines specific and vague references. |
| 2 | Relies on abstract concepts with few technical terms. |
| 1 | Uses marketing language without technical substance. |

**Examples:**
- Written in Rust with GPU-accelerated rendering at 120fps.
- Performance transforms your workflow.

## 2. Natural Syntax
Sentences follow the rhythm of professional conversation.

| Score | Description |
| :--- | :--- |
| 5 | Features varied sentence structures and natural rhythm. |
| 4 | Maintains a natural flow with minor rhythm issues. |
| 3 | Shows occasional structural patterns. |
| 2 | Uses repetitive structures. |
| 1 | Follows a formulaic or robotic cadence. |

**Red flags:** Em dash use. Binary contrast constructions. Parallel lists. Constant sentence lengths.

## 3. Quiet Confidence
Facts demonstrate value without hyperbole.

| Score | Description |
| :--- | :--- |
| 5 | Facts provide evidence for the reader to evaluate. |
| 4 | Statements remain confident with minimal flourish. |
| 3 | Includes occasional hype or superlatives. |
| 2 | Relies on frequent emotional appeals. |
| 1 | Tells the reader how to feel through aggressive marketing. |

**Examples:**
- Zed renders every frame on the GPU. You notice the difference when you scroll.
- Experience the revolutionary speed that transforms how you code.

## 4. Developer Respect
The writing addresses the reader as a technical peer.

| Score | Description |
| :--- | :--- |
| 5 | Assumes technical competence in a peer-to-peer style. |
| 4 | Provides respectful technical depth. |
| 3 | Simplifies concepts unnecessarily. |
| 2 | Includes condescending explanations. |
| 1 | Treats the reader as an uninformed consumer. |

**Examples:**
- Tree-sitter provides incremental parsing. Syntax highlighting updates as you type.
- This technology is fast.

## 5. Information Priority
Key facts lead the communication.

| Score | Description |
| :--- | :--- |
| 5 | Leads with a primary fact or change. |
| 4 | Places important info near the top. |
| 3 | Buries the main point under several sentences. |
| 2 | Requires significant buildup before stating the core message. |
| 1 | Omits or hides key information. |

**Examples:**
- Inline completions now stream token by token. Previously you waited for the full response.
- After months of work we are thrilled to share a new update.

## 6. Specificity
Concrete data supports every claim.

| Score | Description |
| :--- | :--- |
| 5 | Claims are verifiable and specific. |
| 4 | Uses specific language with rare abstractions. |
| 3 | Combines concrete and vague claims. |
| 2 | Relies on abstract benefits. |
| 1 | Claims cannot be verified. |

**Examples:**
- Startup time under 100ms on M1 Macs.
- Fast startup respects your time.

## 7. Voice Consistency
The tone remains unified across all sections.

| Score | Description |
| :--- | :--- |
| 5 | Maintains a coherent voice throughout. |
| 4 | Shows minor tonal shifts. |
| 3 | Features noticeable drift between sections. |
| 2 | Combines competing voices. |
| 1 | Exhibits jarring tonal inconsistency. |

## 8. Earned Claims
Evidence supports assertions.

| Score | Description |
| :--- | :--- |
| 5 | Demonstrable proof backs every assertion. |
| 4 | Assertions are reasonable and mostly verifiable. |
| 3 | Includes unsupported assertions. |
| 2 | Features multiple unverifiable superlatives. |
| 1 | Makes bold claims without backing. |

**Examples:**
- Built by the team behind Atom and Tree-sitter.
- The most advanced editor ever created.

Pass threshold: 32/40. Each criterion requires a score of 4 or higher.

## Decision Rules
- Scores of 4 or higher across all criteria pass the review.
- A score of 3 in any section requires a targeted rewrite.
- A score of 2 or lower requires full reconstruction.
